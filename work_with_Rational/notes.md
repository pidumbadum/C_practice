# 🔬 Внутреннее представление типа `double`

## 📐 Структура IEEE 754 (64 бита)

Тип `double` занимает **8 байт** (64 бита) и состоит из трёх полей:

```
┌─────────────┬──────────────────────┬─────────────────────────────────────┐
│  Знак (1)   │  Экспонента (11)     │  Мантисса (52)                      │
│  63 бит     │  62–52 биты          │  51–0 биты                          │
└─────────────┴──────────────────────┴─────────────────────────────────────┘
```

| Поле | Биты | Размер | Описание |
| :--- | :--- | :--- | :--- |
| **Знак** | 63 | 1 бит | 0 = положительное, 1 = отрицательное |
| **Экспонента** | 62–52 | 11 бит | Степень двойки (со смещением 1023) |
| **Мантисса** | 51–0 | 52 бита | Дробная часть числа |

---

## 🔀 Union для побитового доступа

**`union`** — это специальный тип данных, где все поля делят **одну и ту же ячейку памяти**.

```cpp
union {
    double d;              // Доступ как к числу с плавающей точкой
    unsigned long long b;  // Доступ к тем же байтам как к целому числу
} u;
```

### Как это работает:

1. При создании `u` компилятор выделяет **ровно 8 байт** памяти.
2. И `u.d`, и `u.b` «смотрят» на **одни и те же байты**.
3. Разница лишь в **интерпретации** данных.

```
Память (8 байт): [ 01000001 11001001 00000000 00000000 ... ]
                 └──────────────────────────────────────┘
                                   │
                    ┌──────────────┴──────────────┐
                    │                             │
                u.d (double)              u.b (unsigned long long)
                "3.14159"                 "4614256650576692846"
```

> [!NOTE]
> **`unsigned long long`** — это базовый целочисленный тип данных в C++, гарантированно занимающий 64 бита (8 байт).

## 🛠 Извлечение полей через битовые операции

### 1. Экспонента (11 бит)

```cpp
unsigned long long exp = (u.b >> 52) & 0x7FF;
```

**Что происходит:**

| Шаг | Описание |
| :--- | :--- |
| `u.b >> 52` | Сдвиг вправо на 52 бита (отбрасываем мантиссу) |
| `& 0x7FF` | Маска для получения 11 бит экспоненты |

**Маска `0x7FF`:**
```
0x7FF = 0000 0000 0000 0000 0000 0111 1111 1111
                    ↑ 11 единиц для экспоненты
```

---

### 2. Мантисса (52 бита)

```cpp
unsigned long long mant = u.b & 0xFFFFFFFFFFFFFull;
```

**Что происходит:**

| Шаг | Описание |
| :--- | :--- |
| `& 0xFFFFFFFFFFFFF` | Маска для получения нижних 52 бит |

**Маска `0xFFFFFFFFFFFFF`:**
```
0xFFFFFFFFFFFFF = 0000 1111 1111 1111 ... 1111 1111 1111 1111
                                  ↑ 52 единицы для мантиссы
```

> [!TIP]
> **Суффикс `ull`** говорит компилятору, что это литерал типа `unsigned long long`. Без него число может быть интерпретировано как `int`, что приведёт к ошибке.

---

## 📋 Итоговая шпаргалка

| Операция | Код | Результат |
| :--- | :--- | :--- |
| **Получить экспоненту** | `(u.b >> 52) & 0x7FF` | 11 бит (0–2047) |
| **Получить мантиссу** | `u.b & 0xFFFFFFFFFFFFFull` | 52 бита |
| **Получить знак** | `(u.b >> 63) & 0x1` | 1 бит (0 или 1) |

## ️ Важно

- **Union type punning** — поведение, зависящее от реализации, но широко поддерживаемое в C/C++.
- **`unsigned long long`** гарантирует 64 бита на всех современных платформах.
- Для переносимого кода можно использовать `uint64_t` из `<cstdint>`.

```cpp
#include <cstdint>
union {
    double d;
    uint64_t b;  // Более явный тип
} u;
```
